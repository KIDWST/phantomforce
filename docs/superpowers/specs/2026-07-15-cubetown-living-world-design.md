# CubeTown — Living World Pass — Design

## Context

CubeTown (`app/games/cubetown/`) is a single self-contained IIFE (`cubetown.js`,
~1,474 lines, no build step, no external assets — canvas-drawn primitives and
Web Audio synth only). The world is one fixed 9×9 tile plot with deterministic
seeded terrain (`grass/water/grove/quarry/reed`, generated by a `mulberry32`
PRNG keyed off `state.seed` and never persisted — only the seed is saved).
There's already a day/night cycle (`state.minutes`, `daySegment()`,
`SKY_STOPS` color-lerp, and a firefly ambience system gated on
`rt.reducedMotion`) and three NPCs with time-of-day routines
(`npcSegment()`: work / square / home).

This is the first of several planned expansion passes (a larger brief was
decomposed into: Living World, NPC Depth, World Expansion/biomes, Building &
Expression, Wildlife, Discovery & Secrets, Progression & Atmosphere Polish —
each gets its own spec). This pass adds **seasons, weather, and their
mechanical/NPC/ambience effects**, reusing the existing terrain-generation
philosophy: nothing new is persisted: season and weather are pure functions
of `(state.seed, state.day)`, exactly how `terrain` is already a pure
function of `state.seed`.

## Architecture decision

Season and weather are **derived, not persisted or rolled at runtime** —
computed on demand from `state.seed` and `state.day`, the same way
`genTerrain(state.seed)` already works. This was chosen over (a) rolling and
persisting weather at each day transition, or (b) live host-broadcast
weather, because it requires zero new save fields and zero new multiplayer
sync messages: a guest already receives the host's `seed` via
`adoptSharedSeed` and will independently compute the identical season and
weather. Same free-consistency property `terrain` already has.

## 1. Season system

- `SEASON_LENGTH_DAYS = 7` → 28-day year. `seasonIndex(day) = Math.floor((day - 1) / 7) % 4`.
- Order: **Spring → Summer → Fall → Winter**. Day 1 starts in Spring.
- Each season has its own terrain palette (grass/grove/quarry/reed tint) and
  sky-gradient stops, reusing the existing `lerpColor` / `SKY_STOPS`
  blending technique — parameterized per season instead of hardcoded once.
- **Transition blending**: the last 1.5 days of a season crossfade its
  palette into the next season's via the same lerp helper — no abrupt swap.
- Winter overlays a snow-cover tint on grass tiles (a different
  `TERRAIN_COLOR` entry, selected per season, same drawing code).
- Fall adds falling-leaf ambience (see §4).
- No new persisted fields. `seasonIndex(state.day)` is computed on demand
  everywhere it's needed (drawing, mechanics), like `terrain` already is.

## 2. Weather system

- Types: **Clear, Rain, Fog, Snow**. Snow only occurs in Winter.
- Per-season weighted pools:
  - Spring: Clear 50% / Rain 35% / Fog 15%
  - Summer: Clear 70% / Rain 20% / Fog 10%
  - Fall: Clear 45% / Rain 30% / Fog 25%
  - Winter: Clear 40% / Snow 45% / Fog 15%
- Determinism: `weatherOf(seed, day)` runs `mulberry32(seed ^ (day * 0x9E3779B1))`
  once, picks from that day's season pool. Same day + same seed → same
  weather, for every participant in a shared room and across save/reload.
  Uses the same bitwise-coercion style (`^`, eventual `>>> 0`) already used
  elsewhere in the file — no overflow risk at any day count.
- Weather is fixed for the whole in-game day — one roll per day transition,
  matching the mental model players already have from the day/night clock.
- Visuals:
  - **Rain**: falling streak particles, a desaturating grey wash blended
    into the sky gradient, and temporary puddle-ripple circles that spawn
    only on bare grass tiles with no town piece on them (reusing the
    existing `water` tile's sine-wave shimmer-alpha technique) and fade out.
  - **Fog**: a soft white alpha overlay that increases with tile depth
    (farther tiles fade more — cheap, no real fog-of-war/occlusion logic).
    Also applies a CSS `filter: blur(...)` toggle to the existing `.fish-bar`
    DOM element while fishing in fog (this is a DOM/CSS effect, not canvas).
  - **Snow**: falling particles (slower/heavier drift than the firefly
    ambience) + the Winter ground tint from §1, and footprint marks that
    appear behind the player's last few tiles and fade over ~10 seconds.
  - **Clear**: no overlay.
  - All new particle systems respect `rt.reducedMotion` exactly like the
    existing firefly ambience already does (`if (rt.reducedMotion) return;`)
    — no separate accessibility path to build or maintain.

## 3. Mechanical integration

All effects are multipliers derived from `(season, weather)`; nothing new is
stored in save state.

| Effect | Trigger | Change |
|---|---|---|
| Movement speed | Snow (weather) | `rt.moveDur` +25% (150ms → ~188ms/step) |
| Grain gathering | Rain (weather) | gather duration −20% (faster) |
| Shale gathering | Winter (season) | gather duration +15% (slower) |
| Loom gathering | Fog (weather) | gather duration +15% (slower) |
| Fishing catch zone | Rain or Snow (weather) | zone width +30%, marker speed −15% (easier) |
| Fishing catch zone | Fog (weather) | zone width −20% (harder) |
| Fishing catch zone | Clear | no change (baseline) |
| Spark decay | Winter (season) | `SPARK_DECAY_AMOUNT` 2 → 3 |
| Spark decay | Summer (season) | `SPARK_DECAY_AMOUNT` 2 → 1 |

Dish `spark` values are unaffected in every case — the cost of living
changes, not the value of food, keeping the existing recipe balance intact.

Winter deliberately isn't uniformly punishing: it slows movement and Shale
gathering and raises Spark decay, but Snow weather (45% of winter days)
makes fishing easier — the same bonus Rain gives elsewhere. This was a
deliberate balance fix during design review (an earlier draft had Winter
with only downsides, since Winter's weather pool never rolls Rain).

**NPC schedules**: on Rain or Snow days, any NPC whose current segment would
be `square` goes to `home` instead — residents don't loiter outside in bad
weather. Clear and Fog days are unaffected (fog is atmospheric, not a
"take shelter" trigger). This is a single deterministic override in
`npcSegment()`/`stepNpcs()`, not a new behavior system.

## 4. Ambience & NPC routine depth

- Extends the existing `rt.ambience` firefly system with weather/season-aware
  particle sets (rain streaks + puddles, snow + footprints, fall leaves —
  see §2 for specifics). Same architecture, more particle definitions.
- Each NPC gains one additional daily waypoint — a "leisure" spot near the
  pond at dusk, found via the existing `adjacentGrassTo`/`nearestGrass`
  helpers exactly like `workplace`/`square` already are. Daily loop becomes
  home → work → square → leisure → home (was: home → work → square → home).
- At each stop, NPCs get a small idle variation (subtle facing change /
  short pause-then-shift) so a "parked" NPC reads as doing something. Reuses
  the existing `stepNpcs` interpolation machinery — no new animation system.

## 5. HUD

The existing clock chip (`Day N · time` / `Dawn`) gains a season+weather
line underneath (e.g. `Spring · Rain`) — plain text, consistent with the
HUD's existing icon-free monospace style.

## 6. Save & multiplayer compatibility

Nothing new is persisted and nothing new is synced. `state.day` and
`state.seed` already exist and already sync (a guest already adopts the
host's `seed` via `adoptSharedSeed`). Season and weather are pure functions
of those two already-shared values, so host, guests, and a save/reload all
independently compute the identical season and weather with zero new
messages or fields — the same property `terrain` already has today.

## 7. Testing plan

1. `seasonIndex`/`weatherOf` determinism: same `(seed, day)` in → same
   output out, called repeatedly, and after a simulated save/reload.
2. 28-day cycle wraps correctly (day 29 → Spring again, day 1 of a new year).
3. Mechanical multipliers apply correctly and only under their stated
   trigger, including the Winter+Fog co-occurrence case (Shale slowed by
   season, Loom slowed by weather, simultaneously — both should apply).
4. NPCs correctly skip `square` and go `home` on Rain/Snow days, and are
   unaffected on Clear/Fog days.
5. Puddle ripples never spawn on a grass tile that has a town piece on it.
6. All new particle systems are suppressed when `rt.reducedMotion` is true.
7. No regression to the existing firefly ambience, day/night sky lerp, or
   multiplayer town-sync behavior.

## Non-goals (this pass)

- No new biomes or map expansion beyond the existing 9×9 plot (separate
  "World Expansion" pass).
- No NPC memory/relationship system beyond the new leisure waypoint
  (separate "NPC Depth" pass).
- No new build pieces, wildlife, or discovery/secret events (separate
  passes).
- Weather does not change mid-day; one roll per day transition only.
