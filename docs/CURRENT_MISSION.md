# Current Mission

Transform PhantomForce from an AI tool suite into an operating intelligence.

The current engineering priority is to make the product behave like a real business nervous system: Command, Outcomes, Signals, Decision Cards, Institutional Memory, and Workforce must become the organizing model for the app.

# Focus

## Command

The dashboard command surface should understand intent, use memory, route work, and respond naturally. It should not be a dumb command prompt or a task generator for casual messages.

## Outcomes

The app needs outcome-centered structure. Users should see business progress, not just tabs and tools.

## Signals

Important observations from clients, media, analytics, security, workers, automations, money, websites, and competitors should feed the system as signals.

## Decision Cards

Signals should become concise owner-review cards with evidence, risk, recommended action, and approval state.

## Institutional Memory

Memory must be durable, editable, scoped, and useful. PhantomForce should learn from corrections and stop repeating known mistakes.

## Workforce

Workers, subagents, departments, tools, and automations should be represented as coordinated capability. The UI should show real activity, mapped capacity, and honest status without fake work.

- Command/dashboard.

- Media Lab.

# Engineering Direction

Architecture first. UI second.

Do not redesign simply for appearance. Improve the design when it exposes the architecture more clearly, removes bloat, fixes mobile usability, or makes the owner/operator workflow faster.

Every implementation should answer:

- What real object does this create or update?
- What signal, decision, approval, execution, evidence, memory, or outcome does it connect to?
- What is the source of truth?
- What happens if no real data exists?
- What is gated?
- How does the owner inspect or control it?

# Current Definition Of Done

A change advances the mission when it makes PhantomForce more able to:

- Understand the business.
- Surface the right work.
- Route to the right capability.
- Prepare action.
- Require approval where needed.
- Prove what happened.
- Remember what matters.
- Improve the next run.
